28 /**
29   * Model of OWS Context Documents and provide GeoJson encoding thereof (and maybe AtomXML)
30   * An OWC document is an extended FeatureCollection, where the features (aka entries) hold a variety of metadata
31   * about the things they provide the context for (i.e. other data sets, services, metadata records)
32   * OWC documents do not duplicate a CSW MD_Metadata record, but a collection of referenced resources;
33   *
34   * http://www.opengeospatial.org/standards/owc
35   *
36   * Classically, the WMC documents (Web Map Context documents) were a list of WMS layers for a web map viewer
37   * with a certain context, i.e. title, Bounding Box and a few visualisation properties like scale/zoom,
38   * OWC has superseded that concept into a generic collection of resources:
39   *
40   * We use OWC primarily in the form of collections of case studies, preferably with at least two offerings per entry:
41   * 1) a web visualisable form, e.g.WMS, WFS, SOS ...
42   * 2) a CSW addressable MD_Metadata record according to the resource
43   *
44   * The OWC JSON Encoding is a profile of GeoJSON FeatureCollection, the XML encoding is a profile of Atom/GeoRSS Feed
45   */
46
