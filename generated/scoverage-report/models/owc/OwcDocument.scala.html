32 /**
33   * Model of OWS Context Documents and provide GeoJson encoding thereof (and maybe AtomXML)
34   * An OWC document is an extended FeatureCollection, where the features (aka entries) hold a variety of metadata
35   * about the things they provide the context for (i.e. other data sets, services, metadata records)
36   * OWC documents do not duplicate a CSW MD_Metadata record, but a collection of referenced resources;
37   *
38   * http://www.opengeospatial.org/standards/owc
39   *
40   * Classically, the WMC documents (Web Map Context documents) were a list of WMS layers for a web map viewer
41   * with a certain context, i.e. title, Bounding Box and a few visualisation properties like scale/zoom,
42   * OWC has superseded that concept into a generic collection of resources:
43   *
44   * We use OWC primarily in the form of collections of case studies, preferably with at least two offerings per entry:
45   * 1) a web visualisable form, e.g.WMS, WFS, SOS ...
46   * 2) a CSW addressable MD_Metadata record according to the resource
47   *
48   * The OWC JSON Encoding is a profile of GeoJSON FeatureCollection, the XML encoding is a profile of Atom/GeoRSS Feed
49   */
50 sealed trait OwcFeatureType {
51   val id: String
52   val bbox: Option[Rectangle]
53   val properties: OwcProperties
54 }
